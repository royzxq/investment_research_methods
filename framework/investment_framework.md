## Section 0. 数据输入契约（Data Contract）★ 适配新增

### 0.1 基本约定

### 0.3 数据自检 mini-gate（必过）★ 适配新增

引用数据包前先做 3 项轻校验，**不通过的字段按「该项数据缺失」处理，禁止用旧记忆或估算填充**：

1. **时效一致性**：数据最新交易日与估值时点是否一致 / 可接受；财报报告期是否为最新已披露期。
2. **缺失检测**：关键字段是否显示「—」或「数据为空 / 请人工补充」（常见于 tushare 积分不足导致的北向、披露日历、长周期分位）→ 缺失项转 `[需人工补充]`，不可臆造。
3. **异常值标记**：是否存在可疑数据，例如 OCF/净利剧烈背离、分位 = 100%/0%（疑似常量或样本不足）、扣非与表观异常偏离 → 标注「存疑」，不直接作为 Gate / 熔断的硬依据，需二次核对。

### 0.4 职责边界（保留）

数据包只供「可核验的量价 / 估值 / 财务输入」，**不替代** Precheck 判断、机制归类与 Gate 决策。以下定性项仍由 AI 搜索 + 人工补充，脚本不提供：

- 订单真实性（付费客户 / 合同金额 / 交付周期 / 回款 / 客户集中度 / 公告→转化→回款一致性）
- 海外 AI 大厂 capex 指引方向、订单递延 / 砍单 / 交付放缓信号、与在手订单联动
- 政策落地实锤（文件 / 试点 / 财政资金 / 央企动作 / 反内卷落地）
- Terminal 假设：行业 TAM、份额上限、稳态净利率（三者决定隐含市占率与可达性）
- 国家队（汇金 / 证金）持仓与减持节奏（以官方披露为准，媒体报道不作既成事实）
- 海外流动性紧缩体制的判断输入：美联储点阵图 / 降息预期 / 沃什路径与表态 / 隐含降息 bp
- 代理指标精确值：费城半导体 SOX、港股 VHSI

## 一、分析流程

### 0. 决策原则（Global Rules，保留）

- **Price Map 是最终产品**，所有模块必须回到 P1/P2/T1/T2。
- Gate 仅三种：**通过 / 冻结 / 否决**；冻结 = 禁止新开仓，且必须给出解除条件。
- 结论用区间和分档，不做伪精确。
- 折扣集合 `r_k ∈ {chip, v5, gov, terminal}`，乘法叠加。
- 允许输出「观望 / 回避」，不得强行给买入。
- **必须先做 Precheck Gate**，不通过不得进入估值与 Price Map。
- 买入结论至少建立在以下之一：
  - 盈利兑现
  - 盈利兑现 + 供给出清（反内卷 → PPI 转正）
  - 政策重估（反内卷 / 产业政策） + 盈利兑现
  - AI 成长主线：盈利兑现 + 海外 capex ↔ 国内在手真实订单联动验证通过
  - 负反馈修复 / 高低切：低估值 + 修复证据（利润率修复 / 去库 / 订单恢复 / OCF 转正），经「盈利兑现链」验证
- **纯资金面 / 流动性 / 存款搬家叙事不构成买入条件，也不作为加分项**（宽松流动性 ≠ 买入）。
- **海外流动性紧缩体制为一级硬约束**：持续紧缩（沃什转鹰 / 美国 CPI 破 4% / 点阵图预期加息 / 降息预期归零）是高估值成长的带阈值硬约束。判断主轴 = **盈利能否对冲估值压缩**。本项不把流动性作为回报来源，而是把紧缩作为压缩风险纳入约束。
- **隐含降息前置否决检查**：每只高估值成长标的压测「当前估值是否隐含降息扩张」。若估值依赖已证伪的降息叙事（隐含降息 + 降息预期已归零 / 转加息），作否决式处理，不给买入型 Price Map。
- **追高否决三重过滤**：同时满足三项 → 追高否决（只给跟踪价位、禁止追高新开）：① 拥挤度：公募持仓分位 ≥ 88% 或 板块换手率分位 ≥ 95%；② 估值分位：PE/PB 分位 ≥ 80%；③ 近端业绩：无当季/次季可见业绩，仅远期空间。
- 不得用「政策支持 / 赛道景气 / 远期空间大」替代财报季验证。
- 高估值 / 成长股优先验证：**利润兑现、订单转化、经营现金流质量**。
- AI / 算力 / 应用映射若缺乏真实客户、订单、回款、现金流，不得给完整买入型 Price Map。
- **紧缩下独立成立性检查**：必须判断在持续流动性紧缩 / 无降息 / 估值压缩情景下逻辑是否成立，核心是盈利能否对冲估值压缩。不成立 → 冻结；部分成立 → 不给激进仓位与完整买入型 Price Map。
- **双轴定位强制项**：每只个股同时定位 (a) 主线归属与双韧性（既能脱钩海外 capex 波动 = 盈利端韧性，又能在估值压缩下靠盈利兑现存活 = 估值端韧性）；(b) 拥挤与筹码结构现状（含国家队动向）。两维度都不利者，原则上不给完整买入型 Price Map。
- **滚动事件窗口强制项**：每只个股对当前在册窗口给出前后预案。在册窗口（美联储 / 流动性为一级）：① 美联储议息 / 流动性窗口（沃什路径、美国 CPI、点阵图与降息预期、费半 SOX 行为）；② 海外 AI 大厂 capex / 财报指引窗口；③ 国内反内卷政策落地 / PPI 数据窗口；④ 公司财报窗口（AI 标的重点为中报）。
- **主线消化 ≠ 主线终结**：仅适用于「已验证近端业绩的主线龙头的减仓判断」——单周/单日回调不自动触发减仓或 T1/T2，须先用「海外 capex 指引方向 + 在手真实订单」对回调定性。本条不得作为对高拥挤/高估值/无近端业绩标的追高加仓的理由。
- **阈值是约束而非每周追涨杀跌的开关**：单日/单周噪音不升级为框架级变量。

#### 1.1.1 必答问题（保留）

- 公司当前主要暴露于哪类主矛盾：AI 算力链外需盈利兑现高景气 / 财报季利润·订单·现金流兑现 / 政策重估·反内卷供给出清（PPI 转正）/ 供给冲击与价格重估 / **海外流动性紧缩逆风**（表现为「赚指数不赚钱」）/ 低估值均值回归修复·高低切 / 成本端周期反转 / 内需弱拖累。
- 公司收益逻辑属于：盈利兑现 / 盈利兑现+供给出清 / 政策重估+盈利兑现 / 供给重估 / AI 主线（盈利兑现+海外 capex↔订单联动）/ 负反馈修复（经盈利兑现链）/ 周期反转（经供给重估链或盈利兑现链）/ 均不足。
- **双轴定位必答**：
  - 主线归属与双韧性：主线受益且双韧性强（订单可脱钩 capex + 盈利可对冲压缩）/ 主线受益但脆弱 / 政策供给出清受益 / 负反馈修复·周期反转承接方 / 中性 / 内需端受损。
  - 拥挤与筹码结构现状：清淡 / 中性 / 拥挤 / 拥挤极值（+国家队减持信号）。
  - 双轴判定：双利 / 一利一中 / 一利一损 / 双中 / 一损一中 / 双损。

#### 1.2 硬失败定义（保留，命中任一项即至少冻结）

- 盈利兑现弱，但估值依赖远景叙事
- 订单无法落实到客户/交付/回款
- 现金流差且融资依赖高
- 政策受益无法映射到订单/份额/利润
- capex 明显超前于需求且回收逻辑不清
- 成本传导弱、利润对上游极敏感且无对冲
- 高拥挤题材在财报季缺乏业绩和现金流验证
- AI 商业化缺乏真实付费客户与回款证据
- 双轴定位为「双损」且无明确解套催化
- **高估值成长估值依赖隐含降息扩张，且降息预期已归零/转加息**（隐含降息压测不通过）→ 冻结
- **追高否决三项同时成立且处于追高价位** → 不给买入型 Price Map
- AI 主线标的脱钩假设被证伪（海外 capex 指引转弱 + 在手订单同步转弱）→ 冻结
- 油价上行 30% 情景下 OCF 转负且无对冲（限油价敏感标的）
- 在册事件窗口缺乏任一应对预案

#### 1.3 判定规则（保留）

- 命中 1 项硬失败 → **冻结**；治理/红线 → **否决**。
- 命中 2 项及以上中度风险 → 上调 `r_v5`、`r_chip`，下调 `cap_final`，必要时取消 P2。
- 「拥挤高 + 验证弱」同时成立 → 优先冻结。
- **高估值成长隐含降息扩张 + 降息预期归零** → 冻结（否决式前置）。
- **追高否决三项成立** → 只给跟踪价位，禁止追高新开。
- 「双轴一利一损 + 验证弱」→ 上调 `r_chip` 与 `r_v5`，下调 `cap_final` 30% 以上。
- 「双轴双损」→ 冻结，不给完整买入型 Price Map。
- 全部通过 → 进入估值。

输出：Precheck 状态、主要约束、双轴定位、隐含降息压测结果、追高否决结果、是否允许进入估值、对 P1/P2/cap 的影响。

### 2. 公司机制归类（保留）

归入以下一类并映射参数：

- **稳态现金流型**：看现金流稳态、估值锚、下行保护
- **负反馈修复型**：看利润率修复、去库、订单恢复、现金流转正
- **周期反转型**：看成本、库存、供给、资产负债表
- **供给冲击重估型**：看供给约束、库存变化、价格弹性、利润传导
- **事件驱动重估型**：看政策、治理、资产处置、份额变化能否验证
- **正反馈强化型**：看景气/动量自我强化，须以盈利兑现和现金流支撑为护栏

**机制权重重排**：高权重 = 负反馈修复型；中高权重 = 周期反转型 / 事件驱动重估型；中权重 = 稳态现金流型 / 正反馈强化型 / 供给冲击重估型。

**权重含义澄清**：高权重 = 研究关注度/覆盖权重大，**不等于买入倾向放宽**。负反馈修复升[高] = 加大承接方覆盖，修复证据仍须验证。正反馈降[中] ≠ 放弃 AI；AI 算力链由全面贝塔收敛为**结构阿尔法筛选**，保守约束（高 `d_base`、鉴别清单、追高否决）作护栏保留。高低切至多为短期切换，非成长→价值大切换。

**参数映射规则**：
- 稳态现金流型：标准 `d_base`，保留完整 P1/P2；流动性紧缩体制下进一步定位为**红利/底仓锚**（估值端低敏感、提供下行保护）。
- 负反馈修复型：**升权至高**；典型对象券商/非银/被错杀创新药；修复证据弱则下调 `cap_final`、取消 P2；证据强且双轴非双损时可正常承接；**升权为覆盖加权，买入仍须修复证据兑现**。
- 周期反转型：**升权至中高**；典型对象 PPI/油价资源品；必须先过「正反馈强化 vs 周期反转鉴别清单」；反内卷/供给侧驱动的 PPI 转正不得归此型，应归事件驱动重估型。
- 供给冲击重估型：中权重；供给证据强且利润传导顺畅可保留 P2，否则提高 `r_v5`。
- 事件驱动重估型：**中高权重**；催化弱则下调 `cap_final`，必要时取消 P2；反内卷供给出清归此桶；央企市值管理类可适当放宽，但仍需财报兑现锚点。
- 正反馈强化型：**降权至中**（AI 算力链由全面贝塔→结构阿尔法）；默认提高 `d_base`、下调 `cap_final`，兑现弱则取消 P2；仅逐季兑现 + 订单可见度（1.6T 光模块/存储量价、毛利率、中报）+ 2027 估值合理 者保留买入型 Price Map，并须通过追高否决与隐含降息压测。

**正反馈强化 vs 周期反转鉴别清单**（强制，重点 AI 算力/半导体设备/人形机器人）：
1. 涨价/景气来自真实供需缺口（周期）vs 资本开支驱动的下游虚需求（正反馈）
2. 库存方向：去化中（周期）vs 上下游共振扩产（正反馈）
3. 终端付费客户结构：分散真实付费（周期）vs 同业互锁/算力大厂闭环（正反馈）
4. 现金流验证：OCF 显著好转（周期）vs 利润表好看但 OCF 背离（正反馈）→ `[脚本已供]` OCF·净利比可直接对照
5. 估值分位：低位反转（周期）vs 高位继续抬升（正反馈）→ `[脚本已供]` PE/PB 分位可直接对照

任意 3 项以上指向「正反馈」→ 强制按正反馈强化型处理（即使基本面看似改善）。AI 算力链默认按正反馈强化型，须以证据反证方可归他类；反内卷供给出清归事件驱动重估型。即使归正反馈强化型，在当前体制下亦按**结构阿尔法**对待，须逐季业绩与订单能见度背书方给买入型 Price Map。
